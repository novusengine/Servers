# PostgreSQL base definitions

`Auth.lua`, `Character.lua`, and `World.lua` represent the effective bundle states produced by
applying the legacy `Servers/Resources/Database/Database.sql` file and every timestamped update in
order. Shared definition construction policy lives in `PostgresDefinitions.lua`, outside this
definition tree, so the model loader only discovers actual schema sources here.

The definitions use the Auth, Character, and World ownership split from
`MetaGenPostgreSQLPlan.md`; Shared is intentionally empty. Legacy foreign keys that cross bundle
boundaries are intentionally absent. Sequence-backed `serial` columns become
`GENERATED BY DEFAULT AS IDENTITY` columns in the new baseline.

The generated `PostgresHistory/<bundle>/Migrations/0001_baseline_current_schema.lua` files are the
executable, consolidated baselines for empty databases. They replace the incremental migration
history used while this database work was being developed.

The SQL files under `Baseline/`, along with `harden_inventory_and_map_ids.sql`, are retained as
definition/reference inputs. `MigrationRunner` does not execute them, and their stored functions
and lookup rows are not part of the consolidated baselines. Ownership, tablespaces, dump sequence
state, and development character rows are also not carried forward.

An existing legacy database must be catalog-validated before its baseline is recorded. In
particular, identity conversion and removal of cross-bundle foreign keys require an explicit,
reviewed transition.
